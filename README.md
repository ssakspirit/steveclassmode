# 🎮 Steve Classroom Mode

**마인크래프트 에듀케이션 클래스룸 모드 재현 프로젝트**

마인크래프트 에듀케이션과 **완전히 동일한 WebSocket 프로토콜**을 사용하는 학습보조 도구입니다.
교사가 학생들의 게임 활동을 실시간으로 모니터링하고 제어할 수 있습니다.

---

## 📋 주요 기능

### 1️⃣ 실시간 학생 관리
- ✅ 학생 입장/퇴장 모니터링
- 📍 실시간 위치 추적 (X, Y, Z 좌표 + 차원)
- 💬 채팅 메시지 수신
- 🎯 개별 학생 제어 (텔레포트, 동결, 음소거)

### 2️⃣ 명령 콘솔
- ⚡ 실시간 명령 전송 (`/tp`, `/gamemode`, `/time` 등)
- 🔘 자주 쓰는 명령 프리셋 버튼
- 📜 명령 실행 히스토리

### 3️⃣ 이벤트 로그
- 📋 모든 게임 이벤트 실시간 로그
- 💾 파일 자동 저장 (`logs/` 폴더)
- 🔍 이벤트 타입별 색상 구분

## 🚀 설치 및 실행

### 1단계: 의존성 설치

```bash
npm install
```

### 2단계: TypeScript 빌드

```bash
npm run build
```

### 3단계: 서버 실행

```bash
npm start
```

또는 개발 모드 (빌드 + 실행):

```bash
npm run dev
```

서버가 시작되면 다음과 같이 표시됩니다:

```
🎮 Steve Classroom Mode 서버 시작!
📡 WebSocket 서버: ws://localhost:3000
🌐 웹 클라이언트: http://localhost:3001

마인크래프트 에듀에서 연결하세요:
  /connect ws://localhost:3000
  또는 /wsserver ws://localhost:3000
```

## 🎓 마인크래프트 에듀케이션 연결 가이드

### ✅ 사전 준비 (필수!)

1. **마인크래프트 에듀케이션 실행**
2. **설정 > 일반 > 프로필** 이동
3. **"암호화된 WebSocket 요구" 옵션 OFF** (체크 해제)
   - ⚠️ 이 설정을 끄지 않으면 `ws://` 연결이 차단됩니다!

### 🔌 연결 방법

#### 방법 1: `/connect` 명령 - 프로토콜 없이 (권장)

1. 마인크래프트 에듀에서 멀티플레이어 월드 호스트
2. 채팅창 열기 (T 키)
3. 다음 명령 입력:

```
/connect localhost:3000
```

#### 방법 2: `/connect` 명령 - 프로토콜 포함

```
/connect ws://localhost:3000
```

#### 방법 3: `/wsserver` 명령

```
/wsserver ws://localhost:3000
```

**💡 TIP**: 위 3가지 방법 중 하나만 작동하면 됩니다. 안 되면 다음 방법을 시도하세요!

#### ✅ 연결 성공 확인

- 서버 콘솔에 `✅ 마인크래프트 클라이언트 확인됨!` 메시지 표시
- 웹 브라우저 (`http://localhost:3001`) 우측 상단이 **🟢 서버 연결됨**으로 변경

## 🌐 웹 대시보드 사용법

### 1. 웹 브라우저 열기

```
http://localhost:3001
```

### 2. 대시보드 구성

```
┌─────────────────────────────────────────────────────┐
│  🎮 Steve Classroom Mode    🟢 연결됨  학생: 3명    │
├─────────────────────┬───────────────────────────────┤
│  👥 학생 목록        │  📋 실시간 이벤트 로그         │
│                     │                               │
│  [학생1] 온라인      │  [14:23:45] 학생1 입장        │
│  📍 X:100 Y:64 Z:50  │  [14:24:10] 학생2 채팅...     │
│  🌍 차원: overworld  │  [14:25:30] 학생3 텔레포트    │
│  [TP] [Freeze] [Mute]│                               │
│                     │                               │
├─────────────────────┴───────────────────────────────┤
│  ⚡ 명령 콘솔                                        │
│  [☀️ 낮으로] [🌤️ 날씨 맑게] [📍 모두 모으기]         │
│  /명령어 입력...                    [📤 전송]        │
│  > [14:30:00] /tp @a 0 100 0                        │
└─────────────────────────────────────────────────────┘
```

### 3. 주요 기능

#### 🔘 명령 프리셋 버튼
- **☀️ 낮으로**: `/time set day`
- **🌤️ 날씨 맑게**: `/weather clear`
- **📍 모두 모으기**: `/tp @a 0 100 0`
- **✨ 크리에이티브**: `/gamemode creative @a`
- **⚔️ 서바이벌**: `/gamemode survival @a`
- **🧹 인벤토리 비우기**: `/clear @a`

#### 👤 개별 학생 제어
- **📍 TP**: 학생을 지정 위치로 텔레포트
- **❄️ Freeze**: 학생 이동 제한
- **🔇 Mute**: 학생 채팅 음소거

## 🧪 테스트 시나리오

### 시나리오 1: 기본 연결 테스트

1. ✅ 서버 실행 (`npm start`)
2. ✅ 웹 대시보드 열기 (`http://localhost:3001`)
3. ✅ 마인크래프트 에듀 실행 후 월드 호스트
4. ✅ `/connect ws://localhost:3000` 입력
5. ✅ 웹 대시보드에 자신의 이름이 **학생 목록**에 표시되는지 확인

### 시나리오 2: 실시간 위치 추적

1. ✅ 마인크래프트에서 이동 (걷기, 점프 등)
2. ✅ 웹 대시보드의 **학생 목록**에서 좌표가 실시간 업데이트되는지 확인
3. ✅ **이벤트 로그**에 `PlayerTeleport` 또는 `PlayerTransform` 이벤트 표시 확인

### 시나리오 3: 명령 전송

1. ✅ 웹 대시보드에서 **☀️ 낮으로** 버튼 클릭
2. ✅ 마인크래프트에서 시간이 낮으로 변경되는지 확인
3. ✅ 명령 콘솔 입력창에 `/give @p diamond 64` 입력 후 전송
4. ✅ 마인크래프트에서 다이아몬드 64개가 지급되는지 확인

### 시나리오 4: 채팅 모니터링

1. ✅ 마인크래프트에서 채팅 메시지 입력 (T 키)
2. ✅ 웹 대시보드 **이벤트 로그**에 채팅 내용 표시 확인

### 시나리오 5: 멀티플레이어 테스트

1. ✅ 친구 초대 또는 두 번째 클라이언트 실행
2. ✅ 웹 대시보드 **학생 목록**에 모든 플레이어 표시 확인
3. ✅ 특정 학생 카드에서 **📍 TP** 버튼 클릭
4. ✅ 해당 학생이 (0, 100, 0) 좌표로 텔레포트되는지 확인

## 🔧 기술 스택

- **백엔드**: Node.js + TypeScript
- **WebSocket**: `ws` 라이브러리
- **프로토콜**: 마인크래프트 에듀케이션 WebSocket API (JSON 메시지)
- **프론트엔드**: Vanilla JavaScript (프레임워크 없음)
- **UI**: HTML5 + CSS3 (Flexbox, Grid)

## 🐛 문제 해결

### ❌ "연결 거부됨" 오류

**원인**: 마인크래프트 에듀에서 암호화된 WebSocket만 허용하도록 설정됨

**해결**:
1. 마인크래프트 에듀 **설정 > 일반 > 프로필**
2. **"암호화된 WebSocket 요구"** 체크 해제
3. 게임 재시작

### ❌ 이벤트가 수신되지 않음

**원인**: 마인크래프트가 이벤트 구독을 처리하지 못함

**해결**:
1. `/connect` 명령 재입력
2. 서버 재시작 후 재연결

### ❌ 웹 대시보드가 열리지 않음

**원인**: 3001 포트가 이미 사용 중

**해결**:
1. `server.ts` 파일에서 `httpPort` 변경
2. 서버 재시작
